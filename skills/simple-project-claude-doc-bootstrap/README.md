# simple-project-claude-doc-bootstrap

장기 AI 개발 작업을 위한 최소 프로젝트 문서 체계를 초기화하고 유지하는 claude 스킬입니다.

이 스킬은 문서를 많이 만드는 대신, 꼭 필요한 문서만 유지하면서
AI가 여러 세션에 걸쳐 프로젝트의 목표, 현재 상태, 구조, 작업 규칙을 잃지 않도록 돕습니다.

이 스킬의 기본 철학은 다음과 같습니다.

- 문서는 적을수록 좋다
- 문서는 현재 상태를 반영해야 한다
- 새 문서보다 기존 문서 갱신을 우선한다
- CLAUDE.md는 짧고 공통 규칙만 담는다
- STATUS.md는 긴 로그가 아니라 현재 상태 요약으로 유지한다
- 사용자의 최신 요구사항이 기존 문서보다 우선한다

---

## 관리 대상 문서

기본적으로 아래 4개 문서만 관리합니다.

- `CLAUDE.md`
- `PROJECT.md`
- `STATUS.md`
- `ARCHITECTURE.md`

사용자가 명시적으로 요청하지 않는 한, 추가 문서는 만들지 않습니다.

각 문서의 역할은 다음과 같습니다.

### CLAUDE.md
프로젝트 전반에 공통으로 적용되는 작업 규칙을 기록합니다.

### PROJECT.md
프로젝트 목표, 범위, 제약사항, 성공 기준을 기록합니다.

### STATUS.md
현재 시점 기준의 진행 상황, 다음 작업, 막힌 점을 기록합니다.

### ARCHITECTURE.md
현재 확정된 구조와 주요 설계 결정을 기록합니다.

---

## 이 스킬이 특히 잘 맞는 경우

다음과 같은 프로젝트에서 특히 유용합니다.

- 새 프로젝트를 시작하면서 AI가 장기적으로 이어서 작업해야 하는 경우
- 프로젝트 문서가 거의 없어서 시작점이 모호한 경우
- 기존 문서가 많지만 서로 중복되거나 오래되어 기준 문서가 불분명한 경우
- 여러 세션에 걸쳐 AI가 작업하면서 현재 상태를 자주 놓치는 경우
- 문서 수를 최소화하면서도 맥락 손실을 줄이고 싶은 경우
- 코드보다 문서가 더 빨리 혼란스러워지는 프로젝트를 정리하고 싶은 경우

예를 들면 아래와 같은 상황에 잘 맞습니다.

- 개인 사이드 프로젝트
- 장기적으로 AI와 협업하는 백엔드/프론트엔드 프로젝트
- 점진적으로 기능을 늘려가는 SaaS 프로젝트
- 초기 구조는 있으나 문서 체계가 없는 내부 도구 프로젝트

---

## 이 스킬이 잘 맞지 않는 경우

다음과 같은 경우에는 이 스킬의 기본 철학과 잘 맞지 않을 수 있습니다.

- 매우 엄격한 문서 표준이 이미 정해져 있는 조직 프로젝트
- PRD, ADR, RFC, Runbook, Onboarding 문서 등 별도 문서 체계가 필수인 환경
- 규제, 감사, 인증 대응 때문에 변경 이력 보존이 중요한 프로젝트
- 운영 절차, 장애 대응 절차, 보안 통제 문서가 따로 필요한 프로젝트
- 여러 팀이 협업하며 문서 소비자가 AI보다 사람 조직인 경우

이 스킬은 **문서 최소화와 AI 재개 가능성**에 초점을 둡니다.  
따라서 “문서를 많이 남겨야 하는 조직”보다는 “문서가 너무 많아지면 오히려 방해되는 프로젝트”에 더 잘 맞습니다.

---

## 핵심 원칙

- 문서는 적을수록 좋다
- 새 문서를 만들기 전에 기존 문서에 통합 가능한지 먼저 검토한다
- 문서는 과거 이력을 길게 누적하는 용도가 아니라 현재 상태를 반영하는 용도로 유지한다
- 오래된 내용은 남겨두지 말고 갱신하거나 제거한다
- 사용자의 최신 요구사항이 기존 문서보다 우선한다
- CLAUDE.md에는 모든 세션에 공통 적용되는 규칙만 유지한다
- 의미 있는 작업이 끝난 뒤에는 관련 문서를 반드시 현행화한다

---

## 기대 결과

이 스킬을 잘 사용하면 아래 상태를 목표로 할 수 있습니다.

- AI가 다음 세션에서도 프로젝트를 빠르게 이해할 수 있다
- 어떤 문서가 무엇의 기준 문서인지 명확하다
- 프로젝트 목표, 현재 상태, 구조, 작업 규칙이 서로 겹치지 않는다
- 오래된 계획보다 현재 기준의 문서가 유지된다
- 문서 수가 과도하게 늘어나지 않는다
- 코드와 문서의 불일치가 줄어든다

## 호출 템플릿

아래 템플릿은 바로 복사해서 사용할 수 있습니다.

### 1. 신규 프로젝트 초기화

```text
simple-project-claude-doc-bootstrap 스킬을 사용해서 이 프로젝트의 최소 문서 체계를 초기화해줘.

조건:
- 장기적으로 AI가 여러 세션에 걸쳐 작업할 수 있어야 해
- 문서는 꼭 필요한 것만 만들어줘
- CLAUDE.md, PROJECT.md, STATUS.md, ARCHITECTURE.md만 생성해줘
- CLAUDE.md는 짧고 공통 규칙만 담아줘
- STATUS.md는 긴 작업 로그가 아니라 현재 상태 중심으로 정리해줘

현재 내가 만들려는 프로젝트:
- 프로젝트명: [프로젝트명]
- 목표: [무엇을 만들지]
- 배경: [왜 만드는지]
- 포함 범위: [핵심 범위]
- 제외 범위: [하지 않을 것]
- 제약사항: [기술/일정/운영 제약]

생성 후에는 각 문서의 역할이 서로 겹치지 않게 정리해줘.
```

### 2. 기존 프로젝트 문서 정리

```text
simple-project-claude-doc-bootstrap 스킬을 사용해서 이 저장소의 프로젝트 문서를 최소 구조로 재정리해줘.

목표:
- 장기 AI 작업에 적합한 상태로 문서를 정리하고 싶어
- 문서 수를 줄이고 싶어
- 오래된 내용과 중복 내용을 정리하고 싶어

규칙:
- CLAUDE.md, PROJECT.md, STATUS.md, ARCHITECTURE.md 중심으로 정리해줘
- 새 문서는 꼭 필요한 경우가 아니면 만들지 마
- 기존 문서 내용은 가능한 한 통합하거나 축약해줘
- STATUS.md는 현재 상태 기준으로 다시 써줘
- CLAUDE.md에는 공통 규칙만 남겨줘

정리 후에는
1. 어떤 문서를 유지했는지
2. 어떤 내용을 통합했는지
3. 어떤 문서를 더 이상 기준 문서로 보지 않는지
를 함께 설명해줘.
```

### 3. 현재 코드 기준 문서 생성

```text
simple-project-claude-doc-bootstrap 스킬을 사용해서 현재 저장소 구조와 코드를 기준으로 최소 프로젝트 문서를 작성해줘.

원하는 방식:
- 실제 코드와 디렉토리 구조를 먼저 파악해
- 추측보다는 현재 확인 가능한 사실 위주로 문서를 작성해
- CLAUDE.md, PROJECT.md, STATUS.md, ARCHITECTURE.md만 관리해
- 확인되지 않은 내용은 단정하지 말고 열린 항목으로 남겨줘

중요한 점:
- PROJECT.md에는 현재 프로젝트의 목적과 범위를 정리해줘
- STATUS.md에는 현재 구현된 것과 아직 필요한 것을 구분해줘
- ARCHITECTURE.md에는 실제 코드 기준의 구조만 반영해줘
- CLAUDE.md는 짧게 유지해줘
```

### 4. 작업 후 문서 현행화

```text
방금 진행한 작업 내용을 기준으로 simple-project-claude-doc-bootstrap 스킬 방식으로 프로젝트 문서를 현행화해줘.

원칙:
- 변경된 내용이 있으면 관련 문서만 갱신해줘
- 문서 수는 늘리지 마
- STATUS.md는 현재 상태 중심으로 업데이트해줘
- 구조가 바뀌었으면 ARCHITECTURE.md를 반영해줘
- 목표나 범위가 달라졌다면 PROJECT.md를 반영해줘
- CLAUDE.md는 공통 규칙이 바뀐 경우에만 수정해줘

추가로,
- 이번 작업으로 인해 완료된 것
- 다음에 이어서 할 수 있는 작업
- 아직 불확실한 점
을 STATUS.md에 자연스럽게 반영해줘.
```

### 5. 강한 제약 조건 포함 운영

```text
simple-project-claude-doc-bootstrap 스킬을 사용하되 아래 제약을 강하게 적용해줘.

강한 제약:
- 기본 문서는 CLAUDE.md, PROJECT.md, STATUS.md, ARCHITECTURE.md만 유지
- 새 문서는 내가 명시적으로 요청하지 않으면 만들지 마
- 긴 설명보다 압축된 구조를 선호
- 오래된 내용은 보존보다 갱신을 우선
- STATUS.md를 로그처럼 누적하지 마
- CLAUDE.md에는 넓게 적용되는 규칙만 남겨
- 문서 간 중복을 최대한 제거해

판단 기준:
- 어떤 내용이든 먼저 기존 4개 문서 중 어디에 들어갈지 판단해
- 애매하면 새 파일 대신 STATUS.md 또는 PROJECT.md의 섹션으로 통합해
- 현재 사용자 요구사항을 가장 높은 우선순위로 반영해
```

```text
작업 전에 CLAUDE.md, PROJECT.md, STATUS.md를 읽고 현재 우선순위를 반영해줘.
```

#### 작업 후

```text
방금 변경사항을 기준으로 STATUS.md와 ARCHITECTURE.md를 현행화해줘.
```

#### 방향 변경 시

```text
사용자 요구사항이 바뀌었으니 PROJECT.md와 STATUS.md를 최신 기준으로 다시 맞춰줘.
```

#### 문서 비대화 방지

```text
문서가 늘어나지 않도록 기존 4개 문서 안에서만 정리해줘.
```
