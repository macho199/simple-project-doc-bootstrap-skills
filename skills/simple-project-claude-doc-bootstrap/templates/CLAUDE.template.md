## 작업 원칙
- 사용자의 최신 요구사항을 가장 우선한다.
- 변경은 작고 되돌리기 쉽게 유지한다.
- 코드 변경 후 관련 문서를 함께 갱신한다.
- 불필요한 파일을 만들지 않는다.
- 기존 패턴과 구조를 먼저 파악한 뒤 수정한다.

## 문서 관리 규칙
- 기본 문서는 CLAUDE.md, PROJECT.md, STATUS.md, ARCHITECTURE.md만 유지한다.
- 새 문서를 만들기 전에 기존 문서에 통합 가능한지 먼저 검토한다.
- 오래된 내용은 누적하지 말고 현재 기준으로 갱신한다.
- CLAUDE.md에는 모든 세션에 공통 적용되는 규칙만 유지한다.
- 같은 내용을 여러 문서에 중복 기록하지 않는다.

## 금지 사항
- CLAUDE.md에 작업별 TODO나 임시 계획을 넣지 않는다.
- STATUS.md를 긴 로그처럼 누적하지 않는다.
- 확인되지 않은 내용을 사실처럼 기록하지 않는다.

## 검증 원칙
- 가능한 경우 관련 테스트나 검증을 수행한다.
- 검증이 어렵다면 왜 어려운지 명확히 남긴다.
- 무엇이 바뀌었는지와 남은 과제를 짧게 정리한다.
