사용자 요청:
이미 여러 기능이 진행 중인 API 서비스 저장소를 정리하고 싶다.
진행 중인 작업은 spec 기준으로 재정리하고,
끝난 작업은 완료 상태로 닫고,
운영과 테스트 기준도 지금 구조에 맞게 갱신하라.

기대 동작:
- 기존 `SPECS.md`와 `specs/` 아래 문서를 읽고 현재 상태를 파악한다.
- 오래된 `TASKS.md`는 긴 로그가 아니라 현재 기준 상태로 재작성한다.
- 살아있는 기술 결정이 누락되어 있으면 `DECISIONS.md`를 만든다.
- 외부 API, 이벤트, 파일 형식이 흩어져 있으면 `CONTRACTS.md`로 기준점을 잡는다.
- 공통 테스트 기준이 없다면 `TESTING.md`를 만든다.
- 서비스 운영 기준이 필요하면 `OPERATIONS.md`를 추가한다.
- 문서 역할이 겹치지 않게 중복 내용을 정리한다.
