사용자 요청:
새로운 결제 정산 기능을 구현하려고 한다.
먼저 요구사항, 구현 계획, 작업 체크리스트를 문서로 정리하고,
API 응답 형식이 바뀌면 계약 문서와 테스트 기준도 함께 잡아라.

기대 동작:
- 기존 `AGENTS.md` 또는 `CLAUDE.md`, `PROJECT.md`, `STATUS.md`, `ARCHITECTURE.md`를 먼저 읽는다.
- `SPECS.md`를 만들거나 갱신한다.
- `specs/001-settlement-workflow/` 아래 `SPEC.md`, `PLAN.md`, `TASKS.md`를 만든다.
- 외부 응답 형식이 바뀌면 `CONTRACTS.md`와 `fixtures/contracts/` 예시를 함께 만든다.
- 프로젝트 공통 테스트 기준이 비어 있거나 약하면 `TESTING.md`를 만든다.
- 운영 절차가 필요한 서비스면 `OPERATIONS.md` 필요 여부를 함께 판단한다.
